### 📜 운영 규칙

- **주 5일(월~금)** 1일 1문제 풀이 필수
- 문제 풀이 후 **Commit** & **Pull Request** 로 제출 인증
- 매주 1회 오프라인 리뷰
- 주말(토/일)은 부족한 공부 보충

<br>

### 📂 Repository 구조

`주차/문제명/이름.언어` 형식

```text
Algorithm-Study/
├── README.md
├── week01/
│   ├── BOJ_1000_A+B/
│   │   ├── ko.java
│   │   └── kim.java
│   └── BOJ_2557_Hello/
│       ├── ryu.java
│       └── park.cpp
├── week02/
└── ...

```

<br>

### 💾 참여 방법 (Git Workflow)

1. 이 저장소를 `clone` 합니다.
2. 매주 본인의 브랜치를 생성합니다. (규칙: `이름/주차`)
* 예: `git checkout -b noiskk/week1`


3. 문제를 풀고 커밋합니다.
* **Commit Message**: `[Week1] 이름_문제번호_문제이름`
* 예: `[Week1] noiskk_1000_A+B`


4. 원격 저장소(Origin)로 `push` 합니다.
* 예: `git push origin noiskk/week1`


5. Github에서 **Pull Request(PR)**를 생성합니다.
6. 스터디 모임 전 서로의 코드를 리뷰(Comment)하고 `Merge` 합니다.
